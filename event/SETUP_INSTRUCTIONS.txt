AUTOMATIC BOOKING CANCELLATION - SETUP INSTRUCTIONS
===================================================

This feature automatically cancels bookings that aren't paid within 5 minutes.

STEP 1: Install Dependencies
-----------------------------
pip install celery redis django-celery-beat django-celery-results

STEP 2: Update Django Settings
-------------------------------
Add to your settings.py:

# Celery Configuration
CELERY_BROKER_URL = 'redis://localhost:6379/0'
CELERY_RESULT_BACKEND = 'redis://localhost:6379/0'
CELERY_ACCEPT_CONTENT = ['json']
CELERY_TASK_SERIALIZER = 'json'
CELERY_RESULT_SERIALIZER = 'json'
CELERY_TIMEZONE = 'Asia/Kathmandu'  # Or your timezone

# Optional: Store task results in database
CELERY_RESULT_BACKEND = 'django-db'
CELERY_CACHE_BACKEND = 'django-cache'

Add to INSTALLED_APPS:
    'django_celery_beat',
    'django_celery_results',

STEP 3: Run Migrations
----------------------
python manage.py makemigrations
python manage.py migrate

STEP 4: Install and Start Redis
--------------------------------
# On Ubuntu/Debian:
sudo apt-get install redis-server
sudo systemctl start redis

# On macOS:
brew install redis
brew services start redis

# On Windows:
Download from: https://github.com/microsoftarchive/redis/releases

STEP 5: Start Celery Worker
----------------------------
Open a new terminal and run:
celery -A your_project_name worker --loglevel=info

STEP 6: Start Celery Beat (for periodic cleanup)
-------------------------------------------------
Open another terminal and run:
celery -A your_project_name beat --loglevel=info

STEP 7: Update Project Name
----------------------------
Replace 'your_project_name' in the following files with your actual Django project name:
- your_project_name/celery.py
- your_project_name/__init__.py

HOW IT WORKS
------------
1. When a booking is created, a Celery task is scheduled to run after 5 minutes
2. If the booking is still 'pending' after 5 minutes, it's automatically cancelled
3. Tickets are returned to the event's available pool
4. A periodic cleanup task runs every 10 minutes as a safety net

TESTING
-------
1. Create a booking
2. Wait 5 minutes without paying
3. Check that the booking status changes to 'cancelled'
4. Verify that tickets are returned to the event

PRODUCTION DEPLOYMENT
---------------------
For production, use a process manager like Supervisor or systemd to keep
Celery worker and beat running. Example supervisor config:

[program:celery_worker]
command=/path/to/venv/bin/celery -A your_project_name worker --loglevel=info
directory=/path/to/project
user=your_user
autostart=true
autorestart=true

[program:celery_beat]
command=/path/to/venv/bin/celery -A your_project_name beat --loglevel=info
directory=/path/to/project
user=your_user
autostart=true
autorestart=true
